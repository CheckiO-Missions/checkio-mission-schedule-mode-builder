Вам задано невпорядкований {% if is_js %}масив{% else %} зі списком{% endif %} завдань. Кожне завдання є рядком, який містить початок та кінець завдання і має вигляд "hh:mm-hh:mm". Задачі унікальні. Ваша функція повинна повернути хронологічно відсортований, відфільтрований {% if is_js %}масив{% else %}список{% endif %} завдань, які можна запланувати без перекриття. Для цього вам також надається mode у вигляді {% if is_js %}число{% else %}ціле{% endif %}, яке вказує вам на правило вибору завдань, що не перетинаються.

Режими:

  • 1: найраніший старт серед доступних, (якщо варіантів декілька) найкоротша тривалість;
  • 2найраніший старт з доступних, (якщо кілька варіантів) найдовша тривалість;
  • 3найдовша total тривалість обраних завдань, (якщо декілька варіантів) найбільша кількість виконаних завдань, (якщо декілька варіантів) найраніший;
  • 4: більша кількість виконаних завдань, (якщо кілька варіантів) найдовша total тривалість обраних завдань, (якщо кілька варіантів) найраніша.

При анімації завдання завжди сортуються як найраніший початок + найкоротша тривалість. Кожне завдання розміщується в першому "рядку", де воно не перекривається з попереднім завданням в "рядку". Якщо поточне завдання перекриває всі існуючі "рядки" - починається новий "рядок". Наприклад, для вхідних даних [['09:00-10:00', '09:50-10:10', '10:00-11:00', '09:00-09:20', '10:50-11:10'], 2] анімація (розв'язок) буде виглядати наступним чином:

Вхідні дані: Масив{% if is_js %}n рядків{% else %} зі списком{% endif %} завдань у вигляді рядків.

Вихідні дані: Відсортований відфільтрований {% if is_js %}масив{% else %}список{% endif %} завдань у вигляді рядків.
